# Что готово?
build-lexer — простой сборщик лексера.
# Использование (лексера)!
Создаём каталог (или используем существующий...), создаём файлик 1.layer в нём и пишем там наши строки, которые будут превращаться в токены, используя перенос строки, как разделитель.
Дальше можно создать 2.layer, 3.layer, 4.layer и т.д. по порядку, в которых можно писать расширенные варианты токенов из первого, которые начинаются идентично токенам из первого. Если непонятно, то если в первом слое написано на строке знак равно и во втором написано знак равно, то сгенерированный лексер сможет достаточно эффективно обработать строки «=» (из первого) и «==» (из второго), как соответсвующие токены. Но важно, чтобы они находились на одной строке, считая с начала файла!
В слоях после первого разрешены пустые строки, что будет обозначать пропуск токена — отсутствие расширенного варианта. Если вы хотите альтернативные варианты, то пишите их в первом и последующих слоях отдельно, на новых строках. Конечно же, никакой сортировки нет и всё будет обработано в том же порядке, что вы напишите.
При использовании скрипта на каталоге, он сам возьмёт из него файлы *.layer и создаст/отчистит файл result.c в том же каталоге, чтобы записать в него получившийся код лексера на C.
# Баги (лексера)!
Да-да, в слоях больше первого можно использовать больше строк, чем в первом — недочёт и непорядок.
# Развитие (лексера)!
Возможно, будет сделано указание выходного файла аргументом командной строки вместо жёстко прошитого result.c.
Возможно, будет добавлена поддержка begin.c и end.c, которые будут выставляться в начало и конец выходного файла, окружая код лексера вместо нынешнего временного.
Возможно, будет нечто ещё, но если вам интересно, можете заглянуть в комментарии из кода.
